## 🔧 自动寻找 Skill 机制

### 三个关键节点

| 节点 | 触发时机 | 自动安装的 Skills |
|------|----------|-------------------|
| **产品经理阶段** | 需求分析后 | superpowers-lite, web-search, web-fetch |
| **研发工程师阶段** | 开发前 | superpowers-lite, web-design-guidelines, performance, accessibility-a11y |
| **测试工程师阶段** | 测试前 | superpowers-lite |

### 任务类型识别

系统根据任务名称和描述自动识别类型：
- **web类型**：包含"web、网站、网页、前端、界面、页面、官网"等关键词
- **tool类型**：包含"工具、脚本、程序、自动化"等关键词
- **api类型**：包含"api、接口、后端、服务"等关键词
- **general类型**：其他任务

## 🔄 决策点说明

### 决策点1：迭代确认
```
待确认迭代
   ├── 🔄 用户改为"确认迭代" → 开发中（继续迭代）
   └── ✅ 用户改为"已完成" → 停止任务（结束）
```

### 决策点2：测试结果
```
测试中
   ├── ❌ 测试不通过 → 修复中（自动流转）
   └── ✅ 测试通过 → 待验收（自动流转）
```

## 🚀 快速开始

### 1. 创建新任务
在 Notion 中创建任务，状态设为"未开始"

### 2. 自动流程启动
系统每5分钟自动检查：
- 自动进行产品经理分析
- 自动寻找并安装skills
- 自动开始开发

### 3. 用户介入点
只需在"待确认迭代"状态时决策：
- 继续迭代 → 改为"确认迭代"
- 结束任务 → 改为"已完成"
